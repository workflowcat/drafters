**Кінцеві результати.** [[Subcontractor|Субпідрядник]] передає
[[Contractor|Контрактору]] **усі права інтелектуальної власності**
на матеріали, розроблені або надані Контрактору під час виконання
Послуг ([[Deliverables|результати робіт]]) — **виключно після
повної оплати** всіх винагород та витрат.

**Код з відкритим кодом.** Субпідрядник гарантує, що не включав
і не включатиме жодного open-source коду чи бібліотек під GPL,
AGPL або іншими copyleft-ліцензіями без попередньої письмової
згоди Контрактора, якщо таке включення зобов'язує Контрактора
ліцензувати власне програмне забезпечення далі.

## Критичні моменти

- **«Виключно після повної оплати»** — фраза, яку деякі
  субпідрядники просять прибрати. Не прибирайте без консультації
  з Legal. Це важіль впливу при спорі.
- GPL/AGPL у нашому коді можуть зобов'язати нас відкрити весь
  код, у який вони інтегровані. Для комерційних продуктів це
  смертельно.
- MIT, Apache 2.0, BSD — зазвичай допустимі. Але перевіряйте
  з Legal для кожного нового проєкту.
- При передачі результатів обов'язково фіксуйте факт повної
  оплати — це момент, коли права переходять.
